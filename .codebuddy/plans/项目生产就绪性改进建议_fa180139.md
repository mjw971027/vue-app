## 产品概述

分析现有 Vue 3 + TypeScript + Element Plus 工装管理系统项目，识别其相较于正式可上线项目的核心不足，并给出四条最重要的修改意见及实施方案。

## 四条核心修改意见

### 1. Element Plus 全量引入导致包体积过大，需改为按需引入

当前 `src/main.ts` 全量引入 Element Plus 和所有图标（`import ElementPlus from 'element-plus'` + `for...of Object.entries(ElementPlusIconsVue)`），导致生产包体积膨胀，首屏加载缓慢。正式项目必须按需引入。

### 2. 零测试覆盖，需引入 Vitest 单元测试框架

项目没有任何测试框架、零测试文件、零测试脚本（package.json 无 test 命令）。正式上线项目必须有自动化测试保障核心逻辑（auth 工具函数、API 封装、Store 逻辑）的正确性。

### 3. 安全机制不足，需加固认证存储与传输安全

- Token 存储在 localStorage（XSS 攻击可窃取），应迁移至 httpOnly Cookie 或增加加密防护
- .env 文件被 git 跟踪（.gitignore 未排除 .env），敏感配置暴露
- 密码明文传输无客户端加密
- 无 CSRF 防护、无 CSP 策略

### 4. 生产部署与错误处理缺失，需补齐 404 页面、错误边界与部署配置

- 路由无 404 页面（未匹配路由直接 redirect 到 /，用户无感知）
- 无 Vue 错误边界组件，全局错误处理器仅 console.error
- 无 Dockerfile、无 nginx.conf、无 CI/CD pipeline
- 无路由切换过渡动画与全局 loading 状态

## 技术栈

- 现有：Vue 3 + TypeScript + Vite 8 + Element Plus + Pinia + Vue Router + Axios
- 新增：unplugin-vue-components + unplugin-auto-import（按需引入）、vitest + @vue/test-utils（测试）

## 实施方案

### 修改1：Element Plus 按需引入

**策略**：使用 `unplugin-vue-components` + `unplugin-auto-import` 实现自动按需引入，移除 `main.ts` 中的全量注册。图标组件改为局部导入。

**关键变更**：

- 安装 `unplugin-vue-components`、`unplugin-auto-import`、`unplugin-element-plus`
- `vite.config.ts` 中配置自动导入插件
- `src/main.ts` 移除 `import ElementPlus` 和全局图标注册循环
- 各 `.vue` 文件中使用的图标改为局部 import

**预期效果**：生产包体积减少 40-60%（Element Plus 全量约 800KB+，按需仅引入使用部分）

### 修改2：引入 Vitest 测试框架

**策略**：安装 vitest + @vue/test-utils，优先为安全敏感的核心模块编写测试。

**关键变更**：

- 安装 `vitest`、`@vue/test-utils`、`@pinia/testing`
- `vite.config.ts` 添加 test 配置
- `package.json` 添加 `"test": "vitest"` 脚本
- 编写测试：`src/utils/__tests__/auth.test.ts`（Token 存取、过期检查、JWT 解析、权限判断）
- 编写测试：`src/stores/__tests__/auth.test.ts`（登录/退出状态流转）

### 修改3：安全加固

**策略**：分优先级处理安全问题，短期加固前端可修复部分，长期需后端配合。

**关键变更**：

- `.gitignore` 添加 `.env`、`.env.*`（保留 `.env.example` 作为模板）
- 创建 `.env.example`（不含真实值，仅展示变量名）
- `src/utils/auth.ts` 中 Token 存储增加 AES 加密层（使用 crypto-js 或 Web Crypto API），降低 XSS 窃取后的直接可用性
- 登录时对密码做 SHA-256 哈希（配合后端验证逻辑调整）
- `index.html` 添加 CSP meta 标签基础策略

**注意**：httpOnly Cookie 迁移需后端配合设置 Set-Cookie 响应头，属长期改进项

### 修改4：补齐错误处理与部署配置

**策略**：完善用户体验错误反馈链路 + 提供标准化部署方案。

**关键变更**：

- 新建 `src/views/NotFound.vue`（404 页面），修改路由 `/:pathMatch(.*)*` 指向该页面
- 新建 `src/components/ErrorBoundary.vue`（错误边界组件），捕获子组件渲染错误并展示友好提示
- `App.vue` 添加 `<Transition>` 路由过渡动画，`<router-view>` 包裹
- `src/main.ts` 全局错误处理器集成 Element Plus 的 ElNotification 展示错误提示
- 新建 `Dockerfile`（多阶段构建：node 构建 + nginx 运行）
- 新建 `nginx.conf`（SPA 路由回退 + gzip + 静态资源缓存）
- 新建 `.github/workflows/deploy.yml`（CI/CD 基础模板）
